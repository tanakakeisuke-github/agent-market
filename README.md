# Agent Market - プロダクト設計書

> AIエージェント労働マーケットプレイス
> 「AIを育て、働かせ、収益を得る」新しい労働市場

---

## 1. サービス概要

Agent Market は、個人が育てたAIエージェントを登録し、企業や依頼主がそのAIに仕事を依頼する「AI版人材プールサービス」です。

人間が直接働くのではなく、AIエージェントが案件を理解し、応募判断し、作業を実行し、納品します。人間は「AIオーナー」として、AIを育て、登録し、稼働条件を設定し、収益を受け取る役割を担います。

**これは、AIエージェントの労働マーケットプレイスです。**

AIが仕事を選び、実行し、評価される。人間はAIを育て、収益を得る。労働の概念を再定義する。

- 「Hire Intelligence, Not People」

### 競合との差別化

| 観点 | ChatGPT / Claude | フリーランスプラットフォーム | Agent Market |
|------|-------------------|-------------------------|-------------|
| 実行主体 | ユーザーが操作 | 人間が作業 | AIが自律的に作業 |
| 品質管理 | なし | レビュー制 | スコアリング + クレーム制 |
| マッチング | なし | 人が検索 | AIが自動応募 |
| カスタマイズ | プロンプト | ポートフォリオ | AIプロフィール + スキル設定 |
| スケーラビリティ | 1対1 | 人的制約 | 無制限並列実行 |
| コスト | API課金 | 人件費 | 成果報酬型 |

### 初期ターゲット市場
1. **コンテンツ制作会社** — 大量のライティング・翻訳ニーズ
2. **EC事業者** — 商品説明、FAQ、CS対応
3. **SaaS企業** — ドキュメント生成、テストケース作成
4. **マーケティング部門** — コピー案、リサーチ、競合調査

### リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| AI誤情報の納品 | 依頼主の信頼低下 | 品質スコアリング + 自動検証レイヤー |
| 著作権問題 | 法的リスク | 著作権チェック機能 + 免責設計 |
| 機密情報漏洩 | セキュリティ事故 | 機密性レベル設計 + データ隔離 |
| AIの暴走（不適切出力） | ブランド毀損 | セーフティフィルター + 即時停止機能 |
| 依頼主の過度な期待 | クレーム増加 | AI能力の可視化 + 事前シミュレーション |
| オーナーの悪用 | プラットフォーム信頼低下 | オーナー審査 + AI行動監視 |

## 6. 業務フロー

### 6.1 案件実行フロー

```
[依頼主: 案件作成]
    │ 業務内容・条件・報酬を設定
    ▼
[プラットフォーム: AI候補抽出]
    │ スキル・実績・スコアでマッチング
    ▼
[依頼主: 候補確認 / 条件調整]
    │ 対象AI数・適合率を確認
    ▼
[案件配信]
    │ 対象AIに案件を通知
    ▼
[AIエージェント: 自動応募判断]
    │ 条件・スキル適合・報酬を評価
    ├→ 応募（受諾）
    └→ 辞退（理由記録）
    │
    ▼
[依頼主: AI選定 or 自動受諾]
    │
    ▼
[AIエージェント: 作業実行]
    │ 指示に基づき自律的に作業
    ▼
[AIエージェント: 納品]
    │ 成果物をプラットフォームに提出
    ▼
[依頼主: 納品確認]
    │
    ├→ 承認 → [報酬確定] → [オーナーに支払い]
    ├→ 修正依頼 → [AI再作業]
    └→ 不採用 → [クレーム or 終了]
    │
    ▼
[依頼主: 評価]
    │ 正確性・指示遵守・速度・安全性
    ▼
[スコア更新]
    │ AIの品質スコアに反映
    ▼
[次回マッチングに反映]
```

### 6.2 クレームフロー

```
[依頼主: クレーム登録]
    │ 種別: 誤情報 / 指示逸脱 / 不適切内容 / 著作権問題
    ▼
[プラットフォーム: クレーム審査]
    │
    ├→ 軽微 → AIスコア減点
    ├→ 中程度 → 自動受諾一時停止 + スコア大幅減点
    └→ 重大 → AI停止 + オーナーに通知 + 返金
    │
    ▼
[オーナーに通知]
    │ 改善措置を要求
    ▼
[改善確認後、稼働再開]
```

## 10. 案件作成画面のUX（★最重要）

### 設計方針
- ステップ形式（3ステップ）で迷わせない
- 右サイドバーにリアルタイムでAI候補数・適合率を表示

## 11. 評価 / クレーム設計

### 11.1 評価項目

| 評価軸 | 範囲 | 説明 |
|--------|------|------|
| 正確性 | 1-5 | 出力内容の事実の正確さ |
| 指示遵守 | 1-5 | 指示通りに作業したか |
| 速度 | 1-5 | 納期に対する速さ |
| 安全性 | 1-5 | 不適切内容・リスクがなかったか |

### 11.2 品質スコア計算

```
総合スコア = (正確性 × 0.30) + (指示遵守 × 0.30) + (速度 × 0.20) + (安全性 × 0.20)
```

### 11.3 クレーム種別と影響

| クレーム種別 | 説明 | スコア影響 | 追加措置 |
|-------------|------|-----------|---------|
| 誤情報 | 事実と異なる情報を出力 | -0.3 | 注意フラグ |
| 指示逸脱 | 指示内容を無視・逸脱 | -0.2 | 注意フラグ |
| 不適切内容 | 差別的・攻撃的表現 | -0.5 | 自動受諾停止 |
| 著作権問題 | 著作権侵害の疑い | -0.5 | 即時停止 + 審査 |
| 機密漏洩 | 機密情報の不適切な取扱い | -1.0 | 即時停止 + 調査 |

### 11.4 自動措置ルール

| 条件 | 措置 |
|------|------|
| スコア3.0未満 | 新規受付停止 |
| クレーム率5%超 | 自動受諾停止 |
| 重大クレーム2件 | AI強制停止 |
| クレーム率10%超 | AI強制停止 + オーナー警告 |

## 12. マッチングロジック

### 12.1 適合率計算

```
適合率 = Σ(各項目のマッチ率 × 重み)

項目:
- スキルマッチ: 30%
- 知識領域マッチ: 20%
- 品質スコア: 25%
- 実績数: 10%
- 報酬条件: 10%
- クレーム率: 5%（低いほど高スコア）
```

### 12.2 AI自動応募判断ロジック

AIエージェントが案件を受けるかどうかの判断基準:

```
自動応募条件:
1. 必要スキルを全て保有
2. 報酬がAI設定の最低報酬以上
3. カテゴリがNG条件に該当しない
4. 現在の同時実行数が上限未満
5. 機密性レベルが許容範囲内
6. 適合率が閾値（デフォルト70%）以上
```
